# ИНСТРУКЦИЯ ПО ЗАПУСКУ — cafe

Этот архив содержит полный исходный код проекта **cafe** + готовую docker-compose-сборку.
После выполнения шагов ниже у тебя будет работающий full-stack сайт с локальным AI на машине.

---

## Что нужно поставить ОДИН РАЗ

1. **Docker Desktop** — https://www.docker.com/products/docker-desktop/
   - При установке оставить галку «Use WSL 2» (на Windows)
   - После установки запустить, дождаться зелёного значка в трее
2. **Git for Windows** (для bash-терминала) — https://git-scm.com/download/win
   - Можно и без него, тогда команды выполнять в PowerShell

Больше ничего ставить НЕ НУЖНО — ни Python, ни Node.js, ни Postgres. Всё внутри Docker.

---

## Системные требования

- Windows 10/11, macOS или Linux
- 8 ГБ RAM минимум (16 ГБ комфортно — особенно для AI)
- 15 ГБ свободного места
- Интернет на ПЕРВОМ запуске (чтобы скачать docker-образы и AI-модель ~2 ГБ).
  Дальше — можно работать оффлайн, всё в локальных volume.

---

## Запуск (3 команды)

1. Распаковать этот rar в любую папку без кириллицы и пробелов в пути.
   Например: `C:\projects\cafe\`

2. Открыть **Git Bash** (или PowerShell) в этой папке. В Git Bash это:
   правый клик по папке -> "Open Git Bash here".

3. Выполнить:

```bash
docker compose up -d --build
```

Что произойдёт на первом запуске:
- Docker скачает базовые образы (postgres, ollama, node, python) — 2–4 ГБ, 5–15 минут
- Соберёт backend и frontend из исходников проекта
- Поднимет Postgres, накатит миграции, засеет демо-данные
- Запустит Ollama и автоматически подтянет AI-модель `qwen2.5:3b` (~2 ГБ)
- Поднимет backend и frontend

**Жди ~10–20 минут** на первом запуске. Прогресс смотри командой:

```bash
docker compose logs -f
```

(выйти из логов — `Ctrl+C`, контейнеры останутся работать)

## Управление

```bash
# посмотреть статус контейнеров
docker compose ps

# логи (всех или конкретного сервиса)
docker compose logs -f
docker compose logs -f backend

# остановить (данные ОСТАЮТСЯ — БД сохранена в docker volume)
docker compose stop

# снова запустить (быстро, без билда)
docker compose start

# остановить и удалить контейнеры (volume остаётся)
docker compose down

# ⚠️ ОПАСНО — удалить контейнеры И volume (потеряешь засеянные данные):
docker compose down -v
```

## Если что-то пошло не так

**1. `Cannot connect to the Docker daemon`**
Docker Desktop не запущен. Открой его, дождись зелёного значка, повтори команду.

**2. `bind: address already in use` / `port is already allocated`**
Какой-то другой процесс занял порт (8002, 8081 или 5432).
Найди и закрой его, либо отредактируй `docker-compose.yml` и поменяй маппинг портов
(`"5173:5173"` -> `"5180:5173"`).

**3. Backend перезапускается в цикле (`Restarting (1)`)**
Посмотри логи: `docker compose logs backend | tail -50` — там будет конкретная ошибка.
Чаще всего — БД ещё не успела подняться. Подожди минуту, проверь снова.

**4. AI отвечает 500 / зависает**
Модель qwen2.5:3b ещё качается или грузится в RAM на первом запросе.
Проверь: `docker logs ollama-warmup` — там должно быть `[warmup] готов`.
Первый запрос к AI — 30–60 секунд, последующие — 5–15 секунд.

**5. `no space left on device`**
Освободи 10+ ГБ на диске или почисти неиспользуемые docker-образы:
```bash
docker system prune -a --volumes
```
(⚠️ удалит ВСЁ, включая другие проекты — будь осторожен)

**6. Полностью переустановить с нуля**
```bash
docker compose down -v
docker compose up -d --build
```
