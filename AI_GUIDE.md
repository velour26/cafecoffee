# AI в проекте cafe (Кофеёчек)

## Что это
**AI-бариста кофейни.** Подбирает позицию из меню под настроение / погоду / запрос гостя. Опирается строго на меню (не выдумывает кофе или десертов, которых нет).

## Где смотреть в UI
1. Открой **http://localhost:8081**
2. На **главной странице** (`/`) — блок **«AI-бариста»**
3. Введи свободный запрос (например: «холодно и грустно», «бодрящее перед экзаменом», «что-то сладкое к чаю»)
4. Нажми кнопку — получишь 1–3 рекомендации с объяснением «почему»

## Backend
- **Endpoint:** `POST /api/v1/ai/barista`
- **Файл:** `backend/app/api/v1/endpoints/ai_advisor.py`
- **Модель:** `qwen2.5:3b` (локальная Ollama)
- **Таймаут:** 240 сек

### Пример curl
```bash
curl -X POST http://localhost:8002/api/v1/ai/barista \
  -H "Content-Type: application/json" \
  -d '{"question": "что-то горячее и сладкое"}'
```

## Если AI «недоступен»
При первом запросе модель `qwen2.5:3b` загружается в память (30–60 сек). Бэкенд возвращает `200 OK` с дружественным сообщением вместо ошибки. Init-сервис `ollama-warmup` в compose прогревает модель сразу после старта стека.
